# Maven 使用

## Maven 是什么

Maven 单词的含义为“专家，内行”。Maven 是一个项目管理工具，可以管理的项目类型包括 Java、C#、Ruby、Scala、C++ 等。

## Maven 常见命令

- `mvn help:effective-pom`: 查看生成的 effective pom

## Maven 项目结构

Maven 倡导使用一个共同的标准目录结构，Maven 使用约定由于配置的原则，通常目录结构如下所示：
```
project
├── pom.xml
├── src
│   ├── main
│   │   ├── java【项目的Java源代码】
│   │   ├── resources【项目的资源，例如 property 文件】
│   │   └── webapp
│   │       └── WEB-INF【web应用文件目录，web项目的信息，比如存放 web.xml、本地图片】
│   └── test
│       ├── java【项目的测试类】
│       └── resources【测试用的资源】
└── target【打包输出目录】
    ├── classes【编译输出目录】
    └── test-classes【测试编译输出目录】
```

## Maven 的安装

- 安装 Java
- 下载 Maven 软件包: 下载地址为 https://maven.apache.org/download.cgi
- 解压后，设置环境变量 *MAVEN_HOME*, 在 PATH 中添加 `${MABEN_HOME}/bin`
- 通过 `mvn -v` 验证 maven 是否安装成功

## POM 文件

POM (Project Object Model,项目对象模型) 是 Maven 工程的基本工作单元，是一个 XML 文件，包含了项目的基本信息。用于描述项目如何构建、声明项目依赖等。

执行任务或者目标时，Maven 会在当前目录中查找并读取 POM 文件，获取配置信息，然后执行目标。

POM 中可以指定以下依赖:
- 项目依赖
- 插件
- 执行目标
- 项目构建 profile
- 项目版本
- 项目开发者列表
- 相关邮件列表信息

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <!-- 模型版本 -->
    <modelVersion>4.0.0</modelVersion>

    <!-- 公司/组织唯一标志 -->
    <groupId>com.bfh</groupId>
    <!-- 项目 ID -->
    <artifactId>test</artifactId>
    <!-- 项目版本号 -->
    <version>1.0-SNAPSHOT</version>

</project>
```

所有 POM 文件都要 project 元素和三个必须字段: groupId, artifactId, version，三者组成了项目的坐标。

### 父(Super) POM

父 POM 是 Maven 默认的 POM。所有的 POM 都继承自一个父 POM (无论是否显式定义这个父 POM)。父 POM 包含了一些可以被继承的默认设置。

Maven 使用 effective pom (Super pom 加上工程自己的配置)来执行相关的目标，它帮助开发者在 pom.xml 中做尽可能少的配置，这些配置可以被重写。

使用以下命令查看 Super POM 默认配置：
```sh
mvn help:effective-pom
```
